Este repositorio contiene la implementación de un pequeño sistema de gestion de mascotas.
El problema a resolver en ambos casos es idéntico: registrar y mostrar la información 
básica de una mascota (nombre, especie y edad).

El proyecto se divide en dos enfoques:

Programación Tradicional (programacion_tradicional/): Una solución secuencial que utiliza
variables independientes y funciones para capturar y mostrar los datos por consola.

Programación Orientada a Objetos (programacion_poo/): Una solución modular que emplea una
una clase (Mascota) para encapsular tanto los datos (atributos) como el comportamiento
(métodos), permitiendo instanciar múltiples objetos independientes.

Al desarrollar el mismo sistema bajo dos enfoques distintos, se evidencian diferencias 
operativas y conceptuales significativas que reflejan la evolución misma de los lenguajes
de programación. Mientras que la programación tradicional ofrece una vía rápida y directa
para resolver problemas procedimentales simples, la Programación Orientada a Objetos
proporciona una estructura robusta para modelar sistemas complejos.
La POO no solo organiza el código de manera más natural, sino que sienta las bases conceptuales
para principios más avanzados, preparando el software para adaptarse y escalar en el futuro.
